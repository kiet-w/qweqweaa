Giải Thích Chi Tiết - Module Cart (src/cart)

Module cart là trái tim của hệ thống thương mại điện tử, chịu trách nhiệm quản lý vòng đời giỏ hàng của người dùng.

Tóm tắt các thành phần

Thành phần Mô tả Kiến trúc Flow Liên kết
CartController Cửa ngõ tiếp nhận request, bảo vệ bởi AuthGuard. Auth → Ctrl → UserReq Chi tiết
CartService Logic nghiệp vụ phức tạp, tối ưu hóa với Redis. Svc → Redis → DB Chi tiết
Data Flow Luồng xử lý, Serialization và tác động chéo. Decimal → String → JSON Chi tiết

1. Tầng Giao Tiếp (Controllers): CartController

CartController là cửa ngõ tiếp nhận các HTTP request liên quan đến giỏ hàng.

Phép Thuật Của @UseGuards(AuthGuard) và @Req() user

@UseGuards(AuthGuard)
Xác thực JWT và gắn payload user vào object request.
@Req() user
Trích xuất userId an toàn từ token đã được Guard giải mã.

Các Endpoints Chính

GET/cart
Input: userId (từ token)
Xử lý: Redis getOrSet
Output: CartResponseDto
POST/cart/items
Input: AddCartItemDto
Xử lý: validateProduct, getOrCreateCart, addItem
Output: Updated Cart

2. Tầng Nghiệp Vụ (Services): CartService

CartService là nơi chứa toàn bộ logic nghiệp vụ phức tạp, ứng dụng mạnh mẽ Redis Caching để giảm tải cho Database.

Các Phương Thức Public (Public Methods)

  • getCart(userId: number): Lấy giỏ hàng qua pattern getOrSet của Redis. Nếu Cache Miss, gọi getCartWithItems để query DB và lưu lại cache.
  • addItem(userId, dto):
    • Kiểm tra validateProductAvailability (tồn tại, ACTIVE, đủ stock).
    • Tìm/tạo giỏ hàng ACTIVE qua getOrCreateActiveCart.
    • Cộng dồn số lượng nếu sản phẩm đã có hoặc tạo mới CartItem.
    • Gọi refreshCart để đồng bộ dữ liệu.
  • updateItem(userId, itemId, dto): Bảo mật quyền sở hữu qua findOwnedCartItem, kiểm tra tồn kho và cập nhật số lượng.
  • removeItem(userId, itemId): Xác nhận quyền sở hữu và xóa item khỏi Database.
  • clearCart(userId): Xóa toàn bộ CartItems liên kết với giỏ hàng ACTIVE.

Các Phương Thức Private (Private Helper Methods)

refreshCart(userId)
Xóa cache Redis hiện tại (redisService.del) để lần gọi sau fetch lại dữ liệu mới nhất.
getOrCreateActiveCart(userId)
Tìm giỏ hàng đang ACTIVE hoặc tự động khởi tạo nếu chưa có.
validateProductAvailability(productId, qty)
Kiểm tra tính hợp lệ: Sản phẩm phải tồn tại, ACTIVE và đủ số lượng stock.
findOwnedCartItem(userId, itemId)
Bảo mật cấp độ dữ liệu: Kiểm tra item có thuộc về giỏ hàng ACTIVE của user hiện tại không.
serializeCart(cart)
Tính toán tổng giá trị bằng Prisma.Decimal và convert kết quả sang String.

Lược Đồ Chuyển Đổi (Prisma.Decimal -> String)

Khi lấy dữ liệu từ Database qua Prisma, mọi trường liên quan đến giá tiền đều được biểu diễn bằng Prisma.Decimal để triệt tiêu lỗi dấu phẩy động.

  • Tính toán chính xác: Sử dụng các method .mul() và .plus() để nhân đơn giá và tính tổng.
  • Ép sang String: Trước khi tạo object trả về, mọi Decimal bắt buộc phải được chuyển đổi thành chuỗi qua .toString(). Điều này đảm bảo JSON trả về không bị làm tròn sai lệch, giúp Frontend nhận được con số chính xác tuyệt đối.

4. Tích Hợp Thư Viện Bên Thứ Ba

  • Prisma (Postgres Relations):
    • Ràng buộc duy nhất cartId_productId đảm bảo tính toàn vẹn dữ liệu.
    • Truy vấn lồng nhau (include) để lấy toàn bộ Cây Giỏ Hàng chỉ với một lời gọi.
  • Redis: Tối ưu hiệu suất qua cơ chế Cache Hit/Miss và chủ động xóa cache (Invalidation) khi có thay đổi.
