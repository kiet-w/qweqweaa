1. Tầng Giao Diện Mạng (Controllers): UserController

UserController (src/user/user.controller.ts) là điểm chặn (entry point) đầu tiên cho mọi HTTP request liên quan đến User.

@Controller('users')
Decorator gắn tiền tố /users cho toàn bộ các route bên trong class.
Tác động: Giúp phân tách không gian tên (namespace) rõ ràng giữa các module khác nhau.
👤 User Controller src/user/user.controller.ts
POST /users create()
Input
@Body() data: Prisma.UserCreateInput
Xử lý
Controller gọi userService.create(data).
Output
User object mới được tạo.
GET /users/:id findById()
Input
@Param('id', ParseIntPipe)
Xử lý
Ép kiểu string sang number thông qua Pipe.
Output
Thông tin chi tiết của User.

2. Tầng Logic Nghiệp Vụ (Services): UserService

UserService phụ thuộc vào PrismaService (Database) và RedisService (Cache).

@Injectable()
Đăng ký class vào NestJS DI container như một Provider.
⚙️ User Service src/user/user.service.ts

Các phương thức cốt lõi:

  • create: Ghi DB + redisService.del('all_users') (Cache Invalidation).
  • findByEmail/findById: Dùng findUnique (Độ phức tạp $O(1)$).
  • getAllUsers: Sử dụng redisService.getOrSet và select để ẩn password.

4. Phụ Thuộc Khéo Léo (Dependencies) và Tương Tác với Module Auth

@Inject(forwardRef(() => AuthModule))
Giải quyết Circular Dependency (Phụ thuộc vòng).
Cơ chế: Hoãn việc tham chiếu cho đến khi cả hai module được khởi tạo xong trên bộ nhớ. AuthService sử dụng UserService.findByEmail để xác thực mật khẩu qua bcrypt.
