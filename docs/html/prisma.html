1. Kiến Trúc Toàn Cục (prisma.module.ts)

Module này được đánh dấu bằng decorator @Global(). Đây là một thiết kế quan trọng trong NestJS:

@Global()
Đánh dấu module là toàn cục trong ứng dụng.
Tác động: Khi PrismaModule được import một lần duy nhất vào AppModule, PrismaService sẽ tự động có mặt (injectable) ở khắp mọi nơi trong dự án (User, Catalog, Cart, Orders...) mà không cần phải lặp lại imports: [PrismaModule] ở từng module con.
  • Tái sử dụng liền mạch: Giúp giảm thiểu boilerplate code khi cần truy cập Database.
  • Single Source of Truth: Giảm thiểu việc khởi tạo nhiều instance không cần thiết, giúp quản lý vòng đời (lifecycle) của kết nối Database tại một nơi duy nhất.

2. Đi Sâu Vào PrismaService (prisma.service.ts)

PrismaService kế thừa trực tiếp từ PrismaClient và triển khai interface OnModuleInit. Đây là trung tâm đầu não của việc giao tiếp với cơ sở dữ liệu PostgreSQL. Dưới đây là danh sách chi tiết tất cả các phương thức được định nghĩa bên trong Service này:

2.1. Phương thức khởi tạo: constructor

Đây là thành phần đầu tiên được gọi khi một instance của PrismaService được khởi tạo.

typescriptsrc/prisma/prisma.service.ts
constructor(private readonly configService: ConfigService) {
  const databaseUrl = this.configService.get('DATABASE_URL');
  if (!databaseUrl) {
    throw new InternalServerErrorException('DATABASE_URL is not defined');
  }

  const pool = new Pool({
    connectionString: databaseUrl,
  });
  const adapter = new PrismaPg(pool);
  super({ adapter });
}

Chi tiết luồng xử lý:

  • Lấy cấu hình: Truy xuất biến môi trường DATABASE_URL thông qua ConfigService.
  • Kiểm tra tính hợp lệ: Ném ra lỗi InternalServerErrorException nếu thiếu biến môi trường, chặn lỗi sớm.
  • Tối Ưu Kết Nối Với Driver Adapter:
    • pg Pool: Duy trì một "hồ bơi" kết nối mở sẵn, giúp ngăn chặn triệt để cạn kiệt kết nối (connection exhaustion).
    • Adapter @prisma/adapter-pg: Cầu nối ủy quyền thực thi SQL cho pg Pool thay vì Prisma Engine mặc định.

2.2. Phương thức công khai: onModuleInit()

Hàm này tự động được gọi sau khi module khởi tạo, nhưng trước khi ứng dụng nhận request HTTP đầu tiên.

typescript
async onModuleInit() {
  await this.$connect();
}

Vai trò và cơ chế hoạt động:

  • Eager Connection: Ép Prisma tạo sẵn kết nối ngay lúc boot server (thay vì Lazy Connection mặc định), tránh làm chậm request đầu (cold start).
  • An toàn hệ thống: Nếu cấu hình sai, server báo lỗi và từ chối khởi động ngay lập tức, ngăn chặn "silent failures" khi runtime.

3. Luồng Trả Về Dữ Liệu (Returns/Outputs)

Khi thực thi các phương thức của PrismaService (ví dụ this.prisma.user.findUnique(...)), kết quả trả về là các Promise chứa Dữ liệu thô (Raw Data / Entities).

  • Đích đến: Khối dữ liệu này tuyệt đối không được trả thẳng ra ngoài cho Controller (HTTP response).
  • Nó sẽ đi ngược trở lại Domain Services (ví dụ CatalogService, AuthService, UserService...).
  • Tại các Domain Service, dữ liệu thô từ Prisma sẽ được xử lý logic nghiệp vụ: tính toán giảm giá, lọc bỏ các trường nhạy cảm (như mật khẩu), kiểm tra quyền, v.v., trước khi được ánh xạ (map) sang các DTO.

Ghi chú: PrismaService đóng vai trò thuần túy là Data Access Layer siêu phàm, không can thiệp vào nghiệp vụ.
