1. Controllers (Bộ điều khiển)

File src/orders/orders.controller.ts đóng vai trò là cửa ngõ nhận các HTTP request liên quan đến đơn hàng, xử lý kiểm tra bảo mật ban đầu và chuyển tiếp dữ liệu đến Service.

Các Endpoints

POST /orders/checkout checkout()
Input (vào)
CheckoutDto (thông tin thanh toán)
Logic (trong service)
Tạo Order, Item, trừ stock, đổi status cart.
Output (ra)
OrderResponse object.
GET /orders findOrders()
Input (vào)
userId (từ JWT)
Logic (trong service)
Lấy danh sách đơn hàng đã mua của user.
Output (ra)
Array OrderResponse.
GET /orders/:id findOrderById()
Input (vào)
orderId, userId
Logic (trong service)
Tìm chi tiết 1 đơn hàng cụ thể (bảo mật IDOR).
Output (ra)
Single OrderResponse.

Decorators và Trích xuất Dữ liệu

@UseGuards(AuthGuard)
Bảo vệ API, chỉ cho phép user đã đăng nhập.
Tác động: Gắn thông tin payload của JWT vào request.user.
ParseIntPipe
Biến đổi string ID sang number và validate tính hợp lệ.
Tác động: Ném 400 Bad Request nếu ID không phải là số.

2. Services (Lớp Logic Nghiệp Vụ)

File src/orders/orders.service.ts chứa toàn bộ logic kinh doanh (business logic) cốt lõi của module orders.

Deep dive: Phương thức checkout

Phương thức checkout là trái tim của module này, xử lý quá trình chuyển đổi một giỏ hàng thành đơn hàng thông qua 5 bước bọc trong $transaction.

  1. Lấy Giỏ hàng & Kiểm tra: Đảm bảo giỏ hàng có hàng và sản phẩm vẫn đang ACTIVE.
  2. Tính toán Tổng tiền: Sử dụng Prisma.Decimal để đảm bảo độ chính xác tài chính.
  3. Tạo Đơn hàng: Snapshot giá và tên sản phẩm tại thời điểm mua.
  4. Trừ Tồn kho: Sử dụng updateMany an toàn.
  5. Hoàn tất Giỏ hàng: Đổi trạng thái sang COMPLETED.
typescriptorders.service.ts
await tx.product.updateMany({
  where: {
    id: item.productId,
    stock: { gte: item.quantity }, // ĐIỀU KIỆN THEN CHỐT
  },
  data: { stock: { decrement: item.quantity } }
});
