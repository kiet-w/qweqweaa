1. Cấu Trúc Tổng Quan (Architecture)

Dự án được xây dựng trên framework NestJS, tuân thủ mô hình Modular Architecture. Mỗi tính năng lớn (Auth, Cart, Catalog, Orders) được đóng gói trong một Module riêng biệt bao gồm:

  • Controller: Tiếp nhận Request từ Client, định nghĩa Route và validate dữ liệu đầu vào (DTO).
  • Service: Chứa logic nghiệp vụ chính (Business Logic), tương tác với Database thông qua Prisma.
  • DTO (Data Transfer Object): Quy định cấu trúc dữ liệu gửi lên và phản hồi về.
  • Module: Gắn kết Controller và Service lại với nhau.

Quản Trị Hệ Thống: Cho phép Admin quản lý Danh mục, Sản phẩm và Đơn hàng.

  • Auth: Dùng @Roles(UserRole.ADMIN) và RolesGuard.
  • Catalog & Redis: Khi cập nhật/xóa sản phẩm, gọi RedisService.reset() để hủy cache Catalog.
  • Prisma: Dùng softDelete và include phức tạp.

Hệ Thống Xác Thực: Đăng ký, Đăng nhập, Làm mới Token.

  • User: Phụ thuộc UserService, dùng forwardRef để giải quyết dependency cycle.
  • Security: Dùng HttpOnly Cookies chống XSS.

Danh Mục & Sản Phẩm: Xem danh sách, lọc, tìm kiếm.

  • Tối ưu: Bọc getOrSet quanh Prisma query để tăng tốc độ.

Đơn Hàng & Thanh Toán: Checkout giỏ hàng.

  • ACID: Dùng $transaction để trừ kho và tạo đơn đồng thời.

3. Tương Tác Giữa Các Folder (Cross-Module Communication)

  • Hệ thống Guard & Decorator (src/common): AuthGuard đính request.user để các folder khác dùng.
  • Toàn cục hóa Service (Global Modules): PrismaModule và RedisModule được đánh dấu @Global().
  • Hủy Cache (Cache Invalidation): Folder admin tác động trực tiếp lên catalog qua Redis.

4. Tương Tác Với Prisma & Bảo Toàn Dữ Liệu

  • Atomic Operations: Trừ kho bằng updateMany với điều kiện gte (lớn hơn hoặc bằng).
  • Schema Relation: Xử lý 1-n và n-n mượt mà qua include.
  • Adapter-pg: Dùng Pool từ pg để quản lý kết nối hiệu quả cho Enterprise.

7. Các Kỹ Thuật Nổi Bật (Code Highlights)

  • Transactional Integrity: Sử dụng ACID transaction để bảo vệ dữ liệu tài chính/kho bãi.
  • Performance Optimization: Kết hợp giữa SQL (Prisma) và NoSQL (Redis) để đạt tốc độ phản hồi nhanh nhất.
  • Security Best Practices: Hash mật khẩu bằng bcrypt, bảo mật Token bằng httpOnly.
  • Clean Code: Sử dụng Decorators giúp code ngắn gọn và dễ đọc.
